# COMFIT Storybook

> COMFIT 디자인 시스템 / 공용 UI를 문서화하고, 컴포넌트를 독립적으로 개발·검증하기 위한 공간입니다.

COMFIT Storybook은  
**디자인 시스템을 코드 단위로 관리**하고,  
**팀원 간 UI 컨벤션을 공유**하며,  
**변경 사항의 영향 범위를 빠르게 확인**하기 위한 목적을 가지고 있습니다.

---

## Storybook이란?

Storybook은 UI 컴포넌트를  
**페이지와 분리해서 독립적으로 개발 · 테스트 · 문서화**할 수 있는 도구입니다.

COMFIT 팀에서는 다음과 같은 이유로 Storybook을 도입했습니다.

- 🧩 **컴포넌트 단위 개발**
  - 페이지 구현 전에도 UI 컴포넌트를 단독으로 개발 가능
  - 상태(variant / size / disabled 등)를 명확히 분리해 관리

- 📚 **문서화**
  - 컴포넌트 Props, 사용 예시, 상태를 한 곳에서 확인
  - 신규 팀원 온보딩 시 UI 이해 비용 감소

- ✅ **검증**
  - 다양한 상태를 Story 단위로 고정해두어
  - UI 변경 시 사이드 이펙트 빠르게 확인 가능

---

## COMFIT에서 Storybook을 사용하는 목적

COMFIT 프로젝트는  
페이지 단위보다 **공용 UI / 디자인 일관성**이 중요한 서비스입니다.

Storybook은 단순한 UI 미리보기 도구가 아니라,  
다음 역할을 수행합니다.

### 1️⃣ 디자인 시스템의 기준점

- Button, Input, Modal 등 공용 UI의 **단일 기준(Single Source of Truth)**
- “이 버튼은 어떤 variant가 있는지?”를 코드로 명확히 정의

### 2️⃣ 협업을 위한 커뮤니케이션 도구

- 디자이너 ↔ 프론트엔드 간 UI 논의 기준
- “이 컴포넌트는 Storybook 기준으로 수정합니다”라는 공통 합의

### 3️⃣ 변경 영향 범위 확인

- 공용 컴포넌트 수정 시
  - 어떤 상태가 깨지는지 즉시 확인 가능
- 페이지 QA 전에 UI 레벨에서 1차 검증

## 프로젝트 아키텍처 (Feature-Sliced Design)

COMFIT은 **Feature-Sliced Design(FSD)** 아키텍처를 기준으로 구조를 구성합니다.

```txt
src/
├── app/            # 앱 초기화 / 전역 설정
├── pages/          # 라우팅 단위 페이지
├── widgets/        # 여러 feature를 조합한 UI 블록
├── features/       # 기능 단위 로직/컴포넌트
└── shared/         # 공용 UI / 유틸 / 타입
```

## 📦 Storybook과 FSD의 관계

COMFIT Storybook은 **Feature-Sliced Design(FSD)** 구조를 그대로 반영하여  
어떤 UI를 문서화할지에 대한 기준을 명확하게 정의합니다.

### shared

- **가장 우선적으로 Storybook 문서화 대상**
- 여러 페이지/기능에서 공통으로 사용하는 UI 컴포넌트
- 예시:
  - Button
  - Input
  - Text
  - Badge

> shared 레이어의 컴포넌트는  
> **반드시 Storybook을 통해 사용 방법과 상태를 정의**하는 것을 목표로 합니다.

---

### features / widgets

- **필요 시 문서화**
- 재사용되거나 구조가 복잡한 UI 블록
- 팀 내에서 공통 이해가 필요한 경우 Storybook 대상

> 단순히 특정 페이지에서만 사용하는 UI는  
> Storybook 문서화 대상이 아닙니다.

## 📘 Story 작성 규칙 (가이드)

### 📁 Story 위치

Story 파일은 **컴포넌트와 동일한 디렉토리**에 작성합니다.

```txt
src/shared/ui/button/
├── Button.tsx
└── Button.stories.tsx
```

### 📁 Story 파일 규칙

- `.stories.tsx` 파일은 해당 컴포넌트의 **사용 예시와 상태**를 설명합니다.
- Storybook 전용 디렉토리는 사용하지 않으며,  
  **컴포넌트와 동일한 디렉토리**에 Story 파일을 작성합니다.

---

### 🏷️ Story 네이밍 컨벤션

```ts
title: "Shared/Button";
```

- `Shared / Feature / Widget` 기준으로 카테고리를 구분합니다.
- Storybook 사이드바 구조가 **FSD 구조를 자연스럽게 반영**하도록 구성합니다.

---

### 🎛️ Controls & Args 사용 기준

- `variant`, `size`, `disabled` 등은 **Controls로 제어 가능**하도록 구성합니다.
- 실제 서비스에서 **사용하지 않는 Props는 Story에 포함하지 않습니다.**
- Story는 **“의도된 사용 예시”**만을 표현합니다.

> 여러 Story args는 **초기 예시용**이며,  
> 팀 논의를 통해 최종 컨벤션을 확정할 예정입니다.
